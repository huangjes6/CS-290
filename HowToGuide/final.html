"BONFIRE LIT"

You know, from Dark Souls ... that game we've been using as an example

We're finally there! After a lot of weaving, dodging, hair-pulling, but fortunately no dying, you now have the basics of how to use the Steam Web API. Now you can go on and do some pretty neat things with it. You can make an app that shows all the items and their descriptions that you can get in a game (specifically only Team Fortress 2 and perhaps Dota2 for now). Or you could make an application to have an updated list of all item prices being sold (also only for Team Fortress for now).

I know I just listed two things that are specific for only Team Fortress, but there are a lot more you can do with the Steam Web API. Other examples include going through steam ids to see who owns what game and how long someone spends playing that game, maybe in order to see game popularity. Or you can check a user's friend list and see what the player's friends enjoy playing and compare it to the player's own game list.

Anyways, the point I'm trying to make is, there are various ways you can use the Steam Web API. Now that you know how to make requests and display the results sent from Steam on your own server, all you have to do is create something and make it look nice. Good luck!
